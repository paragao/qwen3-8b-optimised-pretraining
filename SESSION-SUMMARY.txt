## B300 OPTIMIZATION SWEEP (June 17, 2026)

B300's 268 GB memory removes the need for gradient checkpointing, unlocking higher throughput:

| MBS | Seq Len | Grad Ckpt | TFLOP/s/GPU | tok/s (16 GPU) |
|-----|---------|-----------|-------------|----------------|
| 2 | 4096 | Yes | 612 | 199K |
| 2 | 4096 | No | 784 | 255K |
| 4 | 4096 | Yes | 768 | 250K |
| **4** | **4096** | **No** | **976** | **318K** ✅ |
| 8 | 4096 | No | 941 | 306K |
| 4 | 8192 | No | 892 | 145K |
| 8 | 8192 | No | 914 | 149K |

### TP=2 Experiment Result
- TP=2, DP=8, MBS=4: 868 TFLOP/s/GPU, 283K tok/s — **11% worse than TP=1**
- Cause: unnecessary all-reduce communication overhead for a model that fits on one GPU
- Verdict: pure DP is optimal when model fits in single-GPU memory

### B300 vs H200 Final Verdict
- B300 delivers **1.96× the TFLOP/s** and **2.30× throughput** vs H200
- Not 3.4× (theoretical peak ratio) because 8B model is **communication-bound** at this scale
- B300 removes gradient checkpointing requirement → larger MBS → better utilization
- Larger models (70B+) would better exploit B300's 3.4× peak FLOPS advantage

## KEY ARCHITECTURE DECISIONS

### Model Config
- Qwen3-8B: 36 layers, d=4096, 32 Q-heads, 8 KV-heads, FFN=14336 (SwiGLU), vocab=151936
- Precision: BF16
- Seq length: 4096
- Training from scratch (random init, not fine-tuning)

### Parallelism (Final Production Config)
- TP=1, PP=1, DP=16 (pure data parallelism — model fits on one GPU)
- Distributed optimizer (shards Adam states across 16 GPUs)
- Overlapped grad reduce + param gather
- Selective activation recompute (gradient checkpointing)
- MBS=2, GBS=128, grad_accum=4

### Why These Choices
- TP=1: 8.2B fits on one GPU with distributed optimizer + grad ckpt. TP adds comm overhead.
- Distributed optimizer: Without it, Adam states (48 GB fp32) + model (16 GB) + grads (32 GB) = 96+ GB, leaving no room for activations on 141 GB H200.
- Gradient checkpointing: Mandatory. Without it, activations OOM even at MBS=1.
- MBS=2 (not 4): MBS=4 OOMs with Megatron-Core (different memory profile than HF).

## CRITICAL LESSONS LEARNED (Don't Repeat These Mistakes)

### EFA Silent Fallback to TCP
NCCL silently falls back to TCP sockets without these:
1. LD_LIBRARY_PATH must include /opt/amazon/aws-ofi-nccl/lib/
2. NCCL_TUNER_PLUGIN=/opt/amazon/aws-ofi-nccl/lib/libnccl-tuner-aws-ofi.so
3. FI_PROVIDER=efa
4. Symlinks (in this EFA version they already exist at /opt/amazon/aws-ofi-nccl/lib/):
   - libnccl-net-aws-ofi.so → libnccl-net-ofi.so
   - libnccl-tuner-aws-ofi.so → libnccl-tuner-ofi.so
Reference: https://gitlab.aws.dev/smml/benchmarking/dgxc-benchmarks-efa/-/blob/main/container/sglang-efa-b200.Dockerfile

### Slurm/PyXis Gotchas
- Slurm binaries NOT in PATH on compute nodes: always use /opt/slurm/bin/srun, /opt/slurm/bin/scontrol
- Shell vars don't pass into PyXis containers: use --container-env=VAR1,VAR2
- Resolve HEAD_NODE BEFORE srun (scontrol not available inside container):
  export MASTER_ADDR=$(/opt/slurm/bin/scontrol show hostname $SLURM_NODELIST | head -n1)
- For multi-node torchrun: ntasks-per-node=1 (torchrun handles GPU workers)
- For single-node: remove FI_PROVIDER=efa (causes NCCL failure for intra-node)

### enroot import Requires Three Fixes
1. sudo (Docker socket is root:docker, user not in docker group)
2. TMPDIR on /fsx (55 GB NeMo container won't fit in /tmp with 38 GB free)
3. docker buildx use default (image must be in main containerd store)

### Megatron-Core API (v0.13.1 in NeMo 25.07)
- torch.compile INCOMPATIBLE: Set TORCH_COMPILE_DISABLE=1
- Use get_gpt_layer_with_transformer_engine_spec() (NOT local spec)
- BlendedMegatronDatasetBuilder deadlocks with DP — use custom Dataset class
- Tokenizer: use build_tokenizer(get_args())
- Distributed optimizer (--use-distributed-optimizer) is REQUIRED for 8B on H200
- CUDA_DEVICE_MAX_CONNECTIONS=1 only needed for TP (remove for DP-only)
- Qwen3 bridge NOT registered — use GPT model provider with Qwen3 dimensions

### torch.compile — NEVER USE
- Does NOT work with DeepSpeed
- Does NOT work with HF Qwen3 model (multi-node crash)
- Does NOT work in this NeMo container (ldconfig error)
- ALWAYS set TORCH_COMPILE_DISABLE=1
